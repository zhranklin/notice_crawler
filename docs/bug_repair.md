# bug 修复指南

## 适用情形
某个source报告出了问题, 且那个网站的模式比较普通, 可以直接用选择器抓取

## 解决办法
- 安装好java8环境(实在不行的话就6或7)
- 在命令行运行jar包<br>`java -jar xxx.jar`
- 输入指令(详见示例, selector里面的4个字符串分别为:<br>
"内容抓取selector", "日期抓取selector", "新闻url正则表达式", "目录页url")
- 查看结果
- 检查无误后, 在`src/main/scala/com/zhranklin/notice/service/InitVals.scala`文件下添加条目, 格式:<br>
Map(xxx -> (xxx, xxx, xxx, xxx), xxx -> (xxx, xxx, xxx, xxx)), 详见`InitVals.scala`源代码文件(可用`->`代替`→`)

## 检查要点
- 返回的url是否有exception
- 返回的url是否为10条
- 返回的新闻是否有exception
- 返回的新闻是否为10条
- url的顺序是否与网站上的相同
- 新闻的内容是否与网站匹配

请务必仔细检查

## 注意
- 仔细检查结果
- 正则表达式的转义符要用`\\`表示, 一个`\`仅为java/JavaScript语言层面的转转义<br>
语法参照[这里](http://www.runoob.com/java/java-regular-expressions.html)的正则表达式语法章节
- 选择器大体上使用jQuery的语法([参考这里](http://www.jb51.net/article/43996.htm))<br>
但是以jsoup的为准([这里](http://www.open-open.com/jsoup/selector-syntax.htm))<br>
前面链接为教程, 后面链接为扩展语法
